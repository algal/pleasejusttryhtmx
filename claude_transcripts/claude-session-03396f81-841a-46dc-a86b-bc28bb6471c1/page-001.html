# HTMX: Core Concepts & Philosophy

## What Is HTMX?

HTMX is a lightweight JavaScript library (~16kb minified) that extends HTML's capabilities by enabling AJAX, CSS Transitions, WebSockets, and Server Sent Events directly through HTML attributes. It represents a modern approach to building dynamic interfaces while maintaining simplicity.

## Core Philosophy

The library challenges traditional web development constraints:

- Why restrict HTTP requests to only `<a>` and `<form>` elements?
- Why limit triggers to `click` and `submit` events?
- Why confine HTTP methods to GET and POST?
- Why require full-page replacement for updates?

By removing these limitations, "htmx completes HTML as a hypertext."

## Key Features

- **Attribute-driven**: Uses HTML attributes like `hx-post` and `hx-swap` to define behaviors
- **Dependency-free**: No external libraries required
- **Extendable**: Supports custom extensions
- **Efficient**: Has reportedly reduced codebase sizes by 67% compared to React in real-world applications

## The Hypermedia Approach

HTMX embraces REST architectural principles and HATEOAS (Hypermedia As The Engine Of Application State), allowing developers to build modern user interfaces using HTML's inherent power rather than JavaScript-heavy frameworks.

## Quick Example

A button with `hx-post="/clicked"` and `hx-swap="outerHTML"` will POST via AJAX and replace itself with the server response—all without writing JavaScript.

# HTMX Core Documentation Summary

## What is HTMX?

HTMX is a JavaScript library enabling modern browser interactions directly from HTML. As the docs explain, it "allows you to access modern browser features directly from HTML, rather than using javascript."

The fundamental concept extends standard HTML anchor behavior. A regular link performs a GET request and loads the response into the window. HTMX generalizes this pattern to any element, event, HTTP verb, and target.

## Key Attributes

**AJAX Request Methods:**
- `hx-get`, `hx-post`, `hx-put`, `hx-patch`, `hx-delete` — Issue corresponding HTTP requests to specified URLs

**Request Configuration:**
- `hx-trigger` — Specifies which event triggers the request (defaults to natural events: click, change, submit)
- `hx-target` — CSS selector determining where the response loads
- `hx-swap` — Defines how content replaces the target (innerHTML, outerHTML, beforebegin, afterend, etc.)

**Enhanced Features:**
- `hx-confirm` — Prompts user confirmation before issuing requests
- `hx-indicator` — Shows loading spinners during requests
- `hx-boost` — Converts regular links/forms into AJAX requests

## How Requests Work

When triggered, HTMX gathers element values, applies the `htmx-request` class, issues an AJAX request, and swaps the HTML response into the target element. The library expects servers to return HTML fragments rather than JSON.

## Server Response Handling

By default, 204 (No Content) responses do nothing. Error responses (4xx, 5xx) trigger `htmx:responseError` events. Servers can use response headers like `HX-Redirect`, `HX-Refresh`, or `HX-Trigger` to control client behavior.

## Primary Use Cases

- Real-time search and form validation
- Paginated content loading
- Dynamic table updates
- Progressive form submissions
- Interactive UI enhancements without heavy JavaScript

- **Complexity budgets** exhausted before delivering features
